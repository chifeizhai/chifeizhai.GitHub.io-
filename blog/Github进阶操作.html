Github进阶操作

取消变动的暂存（撤销git add .）

执行 git reset HEAD 文件名 来取消暂存。

撤消对文件的修改

执行 git checkout -- 文件名 用来撤销文件的修改。

书签

  • git remote -v 查看书签信息
  • git remote add 书签名 仓库地址 添加新书签
  • git remote set-url 书签名 仓库地址 修改书签地址
  • git remote remove 书签名 删除书签

自定义Git命令

  • git config --global alias.aa 原命令 用aa代替原命令

  • 修改环境变量

分支、分支合并、冲突解决

  • 创建分支 git branch 分支名
  • 切换分支 git checkout 分支名
  • 查看所有分支 git branch -a
  • 创建并立即切换 git checkout -b 分支名
  • 删除分支 git branch -d 分支名

分支合并

例子如下：

  1. git checkout -b bugfix
  2. echo 'fix bug' > index.html
  3. git add .
  4. git commit -m "修复了index.html里的xx bug"
  5. git checkout master (想要合并到哪里就切换到哪里)
  6. git merge bugfix

冲突解决

  1. 执行 git status 查看冲突文件
  2. 输入 git fetch 下载新的分支
  3. 执行 git branch -a 查看新的分支（origin/master）
  4. （可选）执行 git diff 查看文件的变动
  5. 执行 git checkout origin/master 切换到新的分支
  6. 执行 git merge origin master 合并分支
  7. 提示冲突，找到冲突文件开始编辑，搜索 <<<< 找到冲突部分，手动修改并选择需要的内容，删除无关内容。之后保存重新提交
  8. 完成修改提交即可

  • 也可直接执行 git pull origin master 做简单的合并
